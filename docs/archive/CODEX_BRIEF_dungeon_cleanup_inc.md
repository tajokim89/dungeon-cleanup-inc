## 1. 핵심 콘셉트

플레이어는 **몬스터 측 던전 복구 및 위생 관리업체**를 운영한다.

인간 모험가들이 던전을 털고 지나간 뒤, 업체는 던전 주인/마왕성/몬스터 측 의뢰를 받아 현장을 복구한다.

동시에 인간 쪽에는 청소 중 주운 유품, 장비, 시체 등을 유료로 반환하거나 판매해서 부수입을 얻는다.

### 한 줄 기획

> 몬스터 측 던전 복구·위생 관리업체를 운영하며, 인간 모험가 습격 이후의 현장을 청소/복구하고 시체·유품·오염물을 처리해 양쪽에서 돈을 버는 픽셀아트 경영 시뮬.

## 2. 절대 방향성

- 웹게임 아님.
- Godot 기반 픽셀아트 게임.
- SRPG가 본체가 아님.
- **경영 시뮬이 본체**.
- 전술/SRPG 요소는 위험 의뢰에서만 짧게 발생.
- 전투 비중은 5~10% 이하.
- 먼저 예쁜 아트보다 **경영 판단 루프**를 검증한다.

## 3. 게임 비율

- 경영/운영: 80%
- 사건 선택/윤리적 장사 판단: 15%
- 짧은 전술 이벤트: 5%

## 4. MVP 검증 질문

> “플레이어가 복구 품질, 수익, 신뢰, 불법 리스크 사이에서 고민하는 경영 판단이 재밌는가?”

이 질문을 검증하는 것이 MVP의 목적이다.

## 5. MVP 핵심 루프

1. 오늘의 던전 사고 접수
2. 현장 피해 정보 확인
3. 몬스터 직원/장비 배치
4. 처리 방식 선택
5. 정산
   - 자금
   - 마왕성 신뢰도
   - 인간 길드 평판
   - 던전 위생도
   - 불법 처리 위험도
   - 직원 피로/부상
6. 위험 사건이면 짧은 전술 이벤트
7. 하루 종료 보고서
8. 5일 후 업체 평가

## 6. MVP 기간 구조

- 총 5일
- 하루마다 의뢰 3개 제시
- 하루 1~2개 처리
- 5일 후 최종 등급 산정

## 7. 주요 자원

```text
money              자금
hell_trust         마왕성/몬스터 측 신뢰도
human_reputation   인간 길드 평판
hygiene            던전 위생도
illegal_risk       불법 처리 위험도
staff_condition    직원 피로/부상
```

### 실패 조건

- 자금 0 이하
- 마왕성 신뢰도 0 이하
- 직원 전원 부상
- 불법 리스크 100 도달

### 성공 조건

- 5일 생존
- 최종 등급 산정

## 8. 최종 등급 예시

- A: 마왕성 공인 우수 위생업체
- B: 흑자 청소회사
- C: 간신히 버틴 하청업체
- D: 인간 장물 브로커
- F: 폐업 및 던전 출입금지

## 9. 직원 4명

| 이름 | 종족 | 특화 | 약점 |
|---|---|---|---|
| 그릭 | 고블린 | 빠른 청소/잡동사니 회수 | 전투 약함 |
| 멜타 | 슬라임 | 오염/성수/피 처리 | 무거운 물건 약함 |
| 볼그 | 오크 | 시체/잔해 운반 | 함정 처리 약함 |
| 피핏 | 임프 | 함정 복구/장부 조작 | 체력 약함 |

각 직원은 최소한 다음 상태를 가진다.

```gdscript
{
  "id": "grik",
  "name": "그릭",
  "species": "고블린",
  "specialty": "cleanup",
  "stamina": 100,
  "injured": false
}
```

## 10. 장비 5종

| 장비 | 효과 |
|---|---|
| 소독 점액통 | 위생도 손실 감소 |
| 사체 운반 포대 | 시체 반환 수익 증가 |
| 함정 수리 키트 | 복구 품질 증가 |
| 유품 감정 렌즈 | 인간 측 판매/반환 수익 증가 |
| 검은 장부 | 불법 판매 수익 증가, 적발 위험 증가 |

## 11. 의뢰 타입

1. 몬스터 막사 사후 정리
2. 성수 오염 구역 소독
3. 보물방 약탈 피해 복구
4. 미믹 사체 회수
5. 인간 모험가 시체 반환
6. 함정 복구 및 침입로 봉쇄

의뢰는 다음 정보를 가진다.

```gdscript
{
  "id": "holy_water_cleanup",
  "title": "성수 오염 구역 소독",
  "client": "지하 3층 흡혈귀 관리실",
  "pay": 80,
  "risk": 2,
  "tags": ["pollution", "holy", "danger"],
  "description": "성기사 파티가 남긴 성수 폭발 흔적을 제거해야 한다.",
  "requires_tactical_event": false
}
```

## 12. 처리 선택지

각 의뢰마다 2~4개의 처리 선택지를 제공한다.

### 기본 선택지

- 정식 복구
  - 신뢰/위생 증가
  - 수익 낮음
  - 불법 리스크 없음

- 저가 처리
  - 비용 절감
  - 위생/신뢰 손상 가능

- 유품 반환
  - 인간 평판/수수료 증가
  - 몬스터 측 반응은 의뢰 타입에 따라 다름

- 암시장 판매
  - 돈 증가
  - 불법 리스크 증가

- 시체 유료 인도
  - 돈/인간 평판 증가
  - 상황에 따라 마왕성 신뢰도 하락 가능

## 13. 짧은 전술 이벤트

전투는 메인 게임이 아니다.

### 발생 조건

- 인간 잔당 발견
- 폭주한 언데드/미믹 잔해
- 성수 오염 구역에서 직원 구조 필요

### MVP 전술 규칙

- 5x5 또는 6x6 그리드
- 3턴 제한
- 적 전멸보다 목표형
  - 시체 끌고 탈출
  - 직원 보호
  - 인간 정찰병 밀어내기
  - 오염물 회수

## 14. P0 구현 범위

먼저 아래만 구현한다.

1. 5일 루프
2. 매일 의뢰 3개 표시
3. 직원 4명 상태 관리
4. 장비 5종 선택
5. 처리 선택지와 자원 변화
6. 하루 종료 보고서
7. 게임 종료 등급

## 15. P1 구현 범위

P0 이후에 추가한다.

1. 위험 의뢰 1개 전술 이벤트
2. 직원별 특화 보정
3. 불법 리스크 적발 이벤트
4. 인간 시체/유품 반환 수익 구조 강화

## 16. 제외 범위

MVP에서는 하지 않는다.

- 큰 월드맵
- 깊은 SRPG 클래스/스킬트리
- 장비 파밍
- 긴 스토리 컷신
- 다수 던전 바이옴
- 호감도/연애/대화 시스템
- 랜덤 던전 생성
- 본격 전투 밸런싱

## 18. 첫 구현 슬라이스

### Slice 1 — 데이터 모델

목표:
- 직원/장비/의뢰/처리선택지 데이터를 코드로 정의한다.
- 아직 UI는 단순해도 된다.

파일:

```text
data/game_data.gd
scripts/Main.gd
scenes/Main.tscn
```

### Slice 2 — 오늘 의뢰 UI

목표:
- Day 1 표시
- 자원 HUD 표시
- 의뢰 3개 표시
- 의뢰 하나 선택 가능

### Slice 3 — 직원/장비 배치

목표:
- 직원 2명 선택
- 장비 1~2개 선택
- 선택 상태가 화면에 보이게 한다.

### Slice 4 — 처리 선택/정산

목표:
- 선택한 의뢰에 대해 처리 방식 선택
- 자원 변화 적용
- 로그 표시

### Slice 5 — 5일 루프/최종 평가

목표:
- 하루 종료
- 다음 날 의뢰 갱신
- 5일 후 최종 등급 표시

## 19. Codex 작업 지침

- 긴 설계 변경보다 작은 슬라이스 단위로 구현한다.
- 먼저 P0 경영 루프를 만든다.
- 전술 전투는 P0 이후에만 붙인다.
- UI는 일단 기능 우선, 픽셀아트 스킨은 나중.
- 모든 변경 후 Godot에서 실행 가능한지 확인한다.
- 사용자는 Godot GUI로 직접 작업할 예정이므로, 가능한 한 파일 경로와 붙여넣을 코드를 명확히 제시한다.
- 메뉴명은 Godot 영어 UI 기준으로 설명한다.

## 20. 현재까지 사용자 선호/주의사항

- 답변은 짧고 단계적으로.
- 웹게임 전제 금지.
- 픽셀아트 게임이 목표.
- 경영 시뮬 중심.
- SRPG는 살짝만.
- 기획 없이 코드부터 박지 말 것.
- MVP를 디테일하게 쪼개서 구현할 것.
