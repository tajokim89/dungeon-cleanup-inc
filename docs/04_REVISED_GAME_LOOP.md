# 개정 작업 기준서 — 탑다운 사무실 + 던전 작업 + SRPG 전투

## 0. 왜 다시 잡는가

현재 구현된 P0/P1/P2는 의뢰, 직원, 장비, 정산의 기본 구조를 확인하기 위한 메뉴형 프로토타입이다.

하지만 목표 게임은 단순 선택지 게임이 아니라, 플레이어가 사장 캐릭터를 직접 조작하며 사무실과 던전 현장을 오가는 게임이다.

따라서 앞으로의 기준은 아래로 바꾼다.

```text
메뉴형 경영 선택 게임
-> 탑다운 사무실 허브 + 탑다운 던전 작업 + SRPG 전투 게임
```

## 1. 새 핵심 방향

- 플레이어는 회사 사장 캐릭터 1명을 직접 조작한다.
- 사무실에서도 사장으로 움직인다.
- 던전 현장에서도 사장으로 움직인다.
- 직원은 평소에는 편성 자원이며, 이벤트나 전투 시점에 선택/호출되는 전술 자원이다.
- 청소, 회수, 함정 재설치 같은 현장 작업은 탑다운 화면에서 직접 수행한다.
- 전투만 별도 SRPG 턴제 화면으로 전환한다.
- 의뢰/직원/장비 선택은 유지하되, 결과는 버튼 선택이 아니라 실제 현장 행동에서 나온다.

## 2. 새 게임 루프

```text
1. 사무실에서 사장 캐릭터 조작
2. 의뢰 게시판에서 오늘의 의뢰 확인
3. 직원 책상/장비 선반에서 출동 편성
4. 출동문으로 던전 현장 진입
5. 던전에서 사장 캐릭터로 직접 이동
6. E키/Space/마우스로 청소, 회수, 함정 재설치, 유품 수거
7. 위험 상황 발생 시 SRPG 전투 화면으로 전환
8. 전투 결과를 현장 상태에 반영
9. 남은 작업을 계속하거나 출구로 복귀
10. 사무실에서 정산 보고
11. 다음 날 진행
```

## 3. 조작 기준

### 공통

```text
WASD        이동
E / Space   가까운 오브젝트 상호작용
마우스 클릭  대상 선택/작업 지정, UI 선택
Esc         팝업 닫기 또는 일시정지
```

### 사무실

```text
의뢰 게시판   의뢰 확인/선택
직원 책상     직원 편성
장비 선반     장비 선택
장부          자금/평판/리스크 확인
출동문        던전 진입
```

### 던전 현장

```text
오염 타일     청소
잔해          제거
시체 포대     끌기/회수
유품          줍기
고장난 함정   재설치
위험 대상     전투 또는 이벤트 발생
출구          현장 종료
```

## 4. 사장 캐릭터

사장 캐릭터는 항상 플레이어가 직접 조작한다.

초기 MVP에서는 임시 도형/ColorRect/간단한 Sprite2D로 시작해도 된다. 중요한 것은 캐릭터 아트가 아니라 다음 검증이다.

```text
사장이 화면에 보인다.
WASD로 움직인다.
벽/오브젝트와 충돌한다.
상호작용 가능한 대상 근처에서 E키 또는 Space가 작동한다.
사무실과 던전에서 같은 조작감으로 움직인다.
```

## 5. 직원의 역할

직원은 플레이어가 평상시 직접 이동시키는 동료 캐릭터가 아니다.

직원은 다음 순간에 의미를 가진다.

```text
출동 전 편성
현장 작업 보정
특정 이벤트 해결
SRPG 전투 개입
부상/피로/특수 능력
```

예시:

| 직원 | 현장 보정 | 이벤트/전투 역할 |
|---|---|---|
| 그릭 | 잡동사니 수거 속도 증가 | 빠른 이동, 작은 물건 회수 |
| 멜타 | 오염 청소 속도 증가 | 성수/독 오염 피해 완화 |
| 볼그 | 시체/잔해 운반 보정 | 밀치기, 방어, 운반 |
| 피핏 | 함정 재설치 보정 | 함정 해제, 장부 조작, 위험 거래 |

## 6. 전투 구조

전투는 탑다운 현장에서 위험 대상과 접촉하거나 특정 작업을 시작할 때 발생한다.

전투 발생 시 별도 SRPG 화면으로 전환한다.

```text
탑다운 던전 작업
-> 위험 상황
-> BattleEvent 전환
-> 턴제 전투/목표 수행
-> 성공/실패 결과를 던전 현장에 반영
-> 현장 작업으로 복귀
```

전투 목표는 적 전멸만으로 만들지 않는다.

예시 목표:

```text
3턴 동안 버티기
인간 정찰병이 경보를 울리기 전에 제압
시체 포대 보호
오염 핵 파괴
출구까지 후퇴
직원 구조
```

## 7. 정산 구조

기존의 "처리 방식 선택" 중심 정산은 후순위로 밀거나 삭제한다.

정산은 실제 현장 행동 기반으로 계산한다.

```text
청소한 오염 수
제거한 잔해 수
회수한 시체/유품 수
재설치한 함정 수
전투 성공/실패
직원 부상 여부
남긴 오염/잔해
불법 회수품 보유 여부
탈출 여부
```

최종 결과는 다음 자원에 반영한다.

```text
money
hell_trust
human_reputation
hygiene
illegal_risk
staff stamina/injured
```

## 8. 새 씬/스크립트 구조

앞으로 목표 구조:

```text
scenes/
  Main.tscn              전체 상태/씬 전환 관리자
  Office.tscn            사무실 허브
  DungeonTest.tscn       현재 구현된 첫 던전 현장 테스트
  FieldOperation.tscn    던전 현장 작업
  BattleEvent.tscn       SRPG 전투
  ResultReport.tscn      정산 보고

scripts/
  Main.gd
  GameState.gd
  PlayerController.gd
  Office.gd
  DungeonTest.gd
  FieldOperation.gd
  BattleEvent.gd
  Interactable.gd

data/
  GameData.gd
```

현재의 `TacticalEvent.tscn`/`TacticalEvent.gd`는 임시 P1 프로토타입이다. 새 구조에서는 `BattleEvent`로 대체한다.

## 9. 구현 순서

### Slice 0 — 기존 메뉴형 구조 보존/격리

목표:
- 기존 P0/P1/P2 코드를 바로 삭제하지 않는다.
- 새 구조가 동작할 때까지 참고용으로 둔다.
- 이후 `Main.gd`를 씬 전환 관리자 중심으로 정리한다.

### Slice 1 — 사장 캐릭터 + 탑다운 이동

가장 먼저 해야 한다.

목표:
- 사장 캐릭터를 화면에 배치한다.
- WASD 이동을 구현한다.
- 벽/오브젝트 충돌을 확인한다.
- 카메라 또는 화면 중심 구성을 정한다.
- 이동감, 속도, 충돌감이 기본적으로 쓸 만한지 확인한다.

생성/수정 후보:

```text
scenes/Office.tscn
scripts/Office.gd
scripts/PlayerController.gd
```

완료 기준:

```text
Godot Play 시 사무실 방이 나온다.
사장 캐릭터가 보인다.
WASD로 움직인다.
방 밖으로 나가지 않는다.
상호작용 오브젝트 앞에 설 수 있다.
```

### Slice 2 — 사무실 상호작용 오브젝트

목표:
- 의뢰 게시판, 직원 책상, 장비 선반, 장부, 출동문을 배치한다.
- 가까이 가서 E키 또는 Space를 누르면 간단한 패널이 열린다.

완료 기준:

```text
의뢰 게시판 E/Space -> 의뢰 목록 표시
직원 책상 E/Space -> 직원 목록 표시
장비 선반 E/Space -> 장비 목록 표시
출동문 E/Space -> 던전 현장으로 전환
```

### Slice 3 — 던전 현장 탑다운 작업

목표:
- 던전 방 하나를 만든다.
- 사장 캐릭터가 같은 방식으로 움직인다.
- 오염/잔해/시체/함정 오브젝트를 배치한다.
- E키 또는 Space로 작업을 수행한다.

완료 기준:

```text
오염 3개 청소 가능
잔해 1개 제거 가능
시체 포대 1개 회수 가능
함정 1개 재설치 가능
출구에서 현장 종료 가능
```

현재 테스트 구현:

```text
Office 출동문 E/Space -> DungeonTest.tscn 전환
DungeonTest에서 점액 청소/함정 수리/뼈 수거 가능
모든 현장 작업 완료 후 복귀문 E/Space -> Office.tscn 전환
```

### Slice 4 — 현장 결과 정산

목표:
- 실제 작업 결과를 정산에 반영한다.
- 남긴 작업물도 패널티로 반영한다.

완료 기준:

```text
청소한 수에 따라 hygiene/hell_trust 변화
회수품에 따라 money/human_reputation 변화
남긴 오염에 따라 hygiene 하락
현장 종료 후 보고서 표시
```

### Slice 5 — SRPG 전투 전환

목표:
- 던전 현장에서 위험 대상과 접촉하면 `BattleEvent`로 전환한다.
- 전투는 별도 턴제 그리드로 처리한다.
- 전투 후 던전 현장으로 복귀한다.

완료 기준:

```text
위험 대상 접근 -> 전투 시작
턴제 이동/행동 가능
성공/실패 결과 발생
현장으로 복귀
정산에 전투 결과 반영
```

## 10. 지금 당장 다음 작업

다음 큰 작업은 **Slice 4 — 현장 결과 정산**이다.

이유:

```text
사무실 이동과 던전 테스트 작업은 연결됐다.
이제 현장 작업 결과가 자금/신뢰/위생 같은 경영 수치로 돌아와야 한다.
그래야 탑다운 조작과 경영 루프가 같은 게임으로 묶인다.
```

따라서 다음 구현에서는 단순 로그가 아니라 실제 상태 변화를 만든다.

현재 기반:

```text
Office.tscn
DungeonTest.tscn
GameState.gd autoload
PlayerController.gd
Office.gd
DungeonTest.gd
```

현재 완료:

```text
GameState가 day/money/hell_trust/human_reputation/hygiene/illegal_risk를 전역 상태로 가진다.
Office HUD가 GameState 상태값을 표시한다.
회사 장부 E/Space 상호작용이 현재 회사 상태를 표시한다.
DungeonTest 완료 후 복귀하면 자금/신뢰/평판/위생/리스크가 GameState에 반영된다.
Office 복귀 시 하단 메시지에 현장 보고가 표시된다.
```

다음 최소 결과:

```text
의뢰 게시판에서 출동할 현장 선택
직원/장비 선택을 실제 현장 작업에 연결
현장 작업 보상 밸런싱
```

남은 입력 기준:

```text
WASD 이동
E/Space 상호작용
```

```text
픽셀아트 완성

저장/로드

```

## 12. 판단

메인 캐릭터, 즉 사장 캐릭터 생성과 탑다운 움직임을 먼저 개발하는 것이 맞다.

이 게임의 핵심은 이제 "선택지를 누르는 경영"이 아니라 "사장으로 직접 움직이며 회사를 운영하고 현장을 처리하는 게임"이다.

그러므로 다음 기술 검증은 UI가 아니라 조작감이다.
