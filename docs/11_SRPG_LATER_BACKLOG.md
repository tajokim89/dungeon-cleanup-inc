# SRPG 추후 작업 백로그

작성일: 2026-06-29

상태:

- SRPG 전투는 현재 프로토타입 수준에서 일단 멈춘다.
- 당분간 우선순위는 현장 작업, 정산, 의뢰 선택, 직원/장비 편성 쪽이다.
- 이 문서는 나중에 SRPG를 다시 볼 때 이어서 작업하기 위한 별도 기억장소다.

## 현재 SRPG 상태

구현된 것:

- 8x7 전투판
- 사장, 그릭, 멜타 직접 조작
- 이동, 공격, 방어, 지원 명령, 철수, 대기
- 키보드 명령 선택:
  - WASD/방향키로 명령 메뉴 이동
  - E/Space/Enter로 선택
  - 1~6 단축키 유지
- 적은 가장 가까운 아군을 향해 이동하고 인접하면 공격
- 장애물과 경보 장치 오브젝트 표시
- 전투 목표 배너

보류한 것:

- 적 위협 범위 상시 표시
- 목표형 전투 확장
- 전투 UI 아이콘/타일 오버레이 아트
- 전투 코드 분리

## 위협 범위 판단

Fire Emblem Heroes 공식 가이드에는 `Danger Area`가 있다.

참고:

- https://new-guide.fire-emblem-heroes.com/en-GB/feh-1110.html

다만 이 기능은 버튼을 눌러 위험 지역을 확인하는 보조 기능에 가깝다.

현재 프로젝트에는 항상 붉은 범위를 깔아두는 방식이 맞지 않았다. 나중에 다시 넣는다면 아래 중 하나가 낫다.

- 버튼/키로 토글하는 위험 보기
- 특정 적을 선택했을 때만 공격 범위 표시
- 이동 후보 칸에만 작은 경고 아이콘 표시
- 튜토리얼 또는 난이도 보조 옵션으로 제공

## 나중에 할 일

### 1. 목표형 전투

후보:

- 경보 장치 해제
- 3턴 버티기
- 시체 포대 보호
- 오염 핵 제거
- 탈출 타일 도달

첫 구현 추천:

```text
경보 장치 해제 전투
- 목표: 경보 장치 칸에 인접해서 해제
- 승리: 장치 해제
- 패배: 사장 HP 0 또는 철수
```

이유:

- 이미 `OBJECTIVE_CELL`과 경보 장치 표시가 있다.
- 적 전멸이 아닌 목표를 가장 작게 검증할 수 있다.

### 2. 전투 목표/상태 UI 정리

필요한 정보:

- 목표
- 현재 선택 유닛
- 행동 여부
- 남은 적
- 목표 상태

방향:

- 긴 설명보다 짧은 상태 텍스트
- 목표와 로그를 분리
- 명령 선택과 보드 선택 단계를 더 명확히 표시

### 3. 전투와 던전 작업 연결

전투는 별도 모드가 아니라 현장 사고 해결이어야 한다.

예:

```text
경보 장치 조사
-> 잔당 등장
-> 목표형 전투
-> 승리 시 같은 위치로 복귀
-> 장치 해제 완료
```

### 4. 전투 아트/아이콘

나중에 Pixellab 또는 직접 제작으로 만들 것:

- 이동 아이콘
- 공격 아이콘
- 방어 아이콘
- 지원 명령 아이콘
- 철수 아이콘
- 대기 아이콘
- 목표 타일 오버레이
- 선택 유닛 링

### 5. 코드 분리

목표형 전투까지 붙인 뒤 분리한다.

후보:

- `BattleBoard`
- `BattleState`
- `BattleAI`
- `BattleActionMenu`
- `BattleObjective`

지금 당장 분리하지 않는다. 기능이 조금 더 굳은 뒤 자른다.
