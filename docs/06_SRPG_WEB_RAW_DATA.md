# SRPG 웹 리서치 원자료

작성일: 2026-06-28

목적: `던전 클린업 주식회사`의 전투를 단순 전투 팝업이 아니라 SRPG/택틱스 전투처럼 느껴지게 만들기 위한 웹 근거를 모아둔다.

범위:

- 격자 기반 택틱스 RPG / SRPG 전투 구조
- 전투 중 플레이어 의사결정 흐름
- 전투 화면 구성과 HUD 관례
- 현재 1인 보스 전투 프로토타입에 바로 적용 가능한 규칙

출처 품질 기준:

- 공식 사이트, 공식 매뉴얼, 공식 위키를 우선한다.
- 스크린샷 카탈로그는 화면 구성 참고용으로만 쓴다.
- 위키백과류 장르 설명은 보조 맥락으로만 쓴다.

## S1. Fire Emblem Engage 공식 사이트

URL: <https://www.nintendo.com/au/games/nintendo-switch/fire-emblem-engage/>

원자료 관찰:

- 전투를 전략적인 턴제 전투로 설명한다.
- 전장은 격자 기반 전장으로 제시된다.
- 지형, 위치, 유닛 고유 능력, 기술, 무기가 전투 판단 요소로 나온다.
- 무기 상성으로 적을 `break`시키면 적의 반격을 막을 수 있다.
- 즉, 공격 대상 선택은 단순 피해량뿐 아니라 적의 다음 반응까지 바꾼다.

전투 규칙으로 뽑을 점:

- 턴제 전술 전투
- 격자 이동
- 지형과 위치
- 무기/행동 상성
- 반격 가능 여부
- 유닛 능력과 장비 상태가 전투 판단에 연결됨

화면 구성으로 뽑을 점:

- 현재 누구 턴인지 보여줘야 한다.
- 이동, 위치, 공격 선택이 각각 의미 있는 정보로 보여야 한다.
- 유닛을 선택했을 때 어디에 설지, 무엇을 공격할지 판단할 정보가 필요하다.

## S2. Triangle Strategy 공식 Nintendo 가이드

URL: <https://www.nintendo.com/us/whatsnew/tips-for-getting-started-with-triangle-strategy/>

원자료 관찰:

- 화면 하단에 턴 순서 표시가 있다는 점을 언급한다.
- 각 캐릭터 턴은 이동과 행동을 모두 고려한 뒤 종료된다.
- 타일 색상으로 이동 가능 칸, 공격 범위, 위험 범위 같은 의미를 전달한다.
- 타겟 선택 시 적 HP, 명중률 같은 정보를 확인할 수 있다.
- 능력은 Tactical Points 같은 자원을 쓰고, 위치나 높이 조건을 탈 수 있다.
- 고지대, 후방 공격, 협공, 지형 효과, 원소 상호작용이 전술 요소로 언급된다.

전투 규칙으로 뽑을 점:

- 유닛별 턴 순서
- 이동 단계와 행동 단계
- 턴 종료
- 이동/공격/위험 타일 오버레이
- 실행 전 타겟 정보 확인
- 스킬 자원 비용과 조건 체크
- 지형/높이/위치 보정

화면 구성으로 뽑을 점:

- SRPG 화면에는 턴 순서, 현재 행동 유닛, 타일 하이라이트, 타겟 정보, 비활성 스킬 상태가 필요하다.
- 이동 확정 전에 위험 범위를 볼 수 있어야 한다.
- 스킬 버튼은 누른 뒤 실패하는 식이 아니라, 왜 못 쓰는지 미리 보여주는 쪽이 낫다.

## S3. XCOM 2 매뉴얼 - Game Screen

URL: <https://www.feralinteractive.com/en/manual/xcom2/latest/steam/#!spread-15>

원자료 관찰:

- 전술 화면에 임무 목표, 선택 유닛 정보, 무기/탄약 정보, 턴 종료, 행동 버튼이 배치된다.
- 조건이 맞지 않거나 쿨다운 중인 행동은 사용할 수 없게 표시된다.
- 이동은 행동 소모 정도에 따라 다른 색의 경계선으로 보여준다.
- 적 시야, 환경 위험, 엄폐가 전장에 직접 표시된다.
- 공격 타겟팅 시 명중률, 예상 피해, 치명타 확률 같은 전투 예측을 보여준다.
- 행동에는 기본 공격, 경계, 장전, 방어, 상호작용, 탈출, 들기/내리기, 특수 기술, 턴 종료가 포함된다.

전투 규칙으로 뽑을 점:

- 행동 경제
- 조건 미충족/쿨다운에 따른 비활성 행동
- 전술 이동 범위 표시
- 위험/시야/환경 정보 표시
- 공격 예측
- 방어/대기형 행동
- 목표 상호작용 행동
- 탈출/철수 행동

화면 구성으로 뽑을 점:

- 행동 버튼은 하단 또는 측면 고정 바에 두면 읽기 좋다.
- 각 행동은 아이콘, 이름, 사용 가능 상태, 조건을 가져야 한다.
- 전투 화면에는 적 HP뿐 아니라 현재 목표가 보여야 한다.
- 공격 실행 전 예상 결과를 보여주는 것이 전술 판단의 핵심이다.

## S4. Wesnoth 공식 위키 매뉴얼

URL: <https://wiki.wesnoth.org/ItalianWesnothManual#Gameplay>

원자료 관찰:

- 유닛 선택 시 도달 가능한 칸과 도달 불가능한 칸이 다르게 보인다.
- 지형에 마우스를 올리면 이동 비용과 방어 값을 확인할 수 있다.
- 적 인접 칸에는 ZOC, 즉 통제 구역 개념이 적용된다.
- 기본 설명에서는 공격이 주로 인접 전투 흐름으로 설명된다.
- 방어자는 호환되는 사거리의 무기가 있으면 반격한다.
- 유닛 상태는 색상 구슬로 표시된다. 이동 가능, 공격 가능, 행동 종료 여부를 시각적으로 구분한다.

전투 규칙으로 뽑을 점:

- 이동 가능 칸 하이라이트
- 지형 정보 확인
- 적 인접 제약
- ZOC
- 공격/반격 교환
- 행동 완료 상태 표시

화면 구성으로 뽑을 점:

- 전술 화면의 중심은 지도지만, 타일 정보는 hover/선택으로 바로 볼 수 있어야 한다.
- 유닛이 이미 행동했는지 메뉴를 열지 않고도 보여야 한다.
- 적과 인접했을 때 선택 가능한 행동이 바뀌어야 한다.

## S5. Tactics Ogre: Reborn 전투 시스템 기사

URL: <https://www.gematsu.com/2022/10/tactics-ogre-reborn-details-unit-development-charms-battle-updates-balance-adjustments-shop-improvements-classes-and-characters>

원자료 관찰:

- 전투 전 파티 편성 화면에서 유닛을 준비할 수 있다.
- `Scout` 명령으로 전투 전 지형과 적 정보를 확인할 수 있다.
- 스킬, 소비품, 마법, 장비 구성이 전투 전 전략에 포함된다.
- 전장에 버프 카드가 나타나며, 특정 칸으로 이동할 이유를 만든다.
- 일반 승리 조건 외에 보너스 목표가 존재한다.
- 궤도 예측 기능으로 원거리/투사체 타겟팅 실수를 줄인다.
- 적 AI는 지형과 플레이어 움직임을 고려하는 식으로 설명된다.

전투 규칙으로 뽑을 점:

- 전투 전 정찰
- 파티/장비/스킬 준비
- 명시적인 승리 조건과 보너스 목표
- 전장 위 수집/버프 포인트
- 타겟팅 궤도 예측
- 지형 고려 AI

화면 구성으로 뽑을 점:

- 전투 시작 전에 목표를 명확히 보여줘야 한다.
- 타겟팅 미리보기는 조작 실수를 줄인다.
- 현재 프로젝트는 파티 배치는 미뤄도 되지만, 전투 진입 시 목표와 선택 목표는 보여주는 편이 좋다.

## S6. Interface In Game - XCOM 2 UI 스크린샷

URL: <https://interfaceingame.com/games/xcom-2/>

원자료 관찰:

- UI 상태가 피해, 적 타겟, 게임 오버, 아이콘, 인벤토리, 지도, 이동, 내비게이션, 옵션, 일시정지, 유닛, 경고 등으로 분류된다.
- 전술 게임은 고정 HUD 하나만 있는 것이 아니라 상태별 UI가 반복해서 필요하다는 점을 보여준다.

전투/UI 상태로 뽑을 점:

- 이동 상태
- 타겟팅 상태
- 경고 상태
- 유닛 정보 상태
- 피해/결과 상태
- 메뉴/일시정지/옵션 상태

화면 구성으로 뽑을 점:

- 첫 프로토타입부터 상태별 UI를 나눠두는 편이 좋다.
- 처음에는 단순해도, 이동 선택 UI와 공격 선택 UI는 분리되어야 한다.

## S7. Interface In Game - Wargroove UI 스크린샷

URL: <https://interfaceingame.com/games/wargroove/>

원자료 관찰:

- UI 상태가 행동, 캐릭터, 피해, 아이콘, 지도, 이동, 내비게이션, 자원, 스킬, 유닛, 승리 등으로 분류된다.
- 픽셀/격자 전술 게임에서도 간결한 UI 상태 구분이 중요하다.

전투/UI 상태로 뽑을 점:

- 선택 유닛 상태
- 이동 범위 상태
- 행동 선택 상태
- 스킬 상태
- 피해 상태
- 승리 상태

화면 구성으로 뽑을 점:

- 지도가 주 플레이 공간이면 큰 패널보다 compact한 정보 UI가 더 낫다.
- 피해 표시와 승리 표시도 독립 상태로 필요하다.

## S8. Tactical Role-Playing Game 장르 개요

URL: <https://en.wikipedia.org/wiki/Tactical_role-playing_game>

원자료 관찰:

- 장르 설명은 RPG 성장 요소와 전술적 전투를 결합한 형태로 설명한다.
- 메뉴형 RPG 전투와 다른 핵심은 위치, 이동, 전장 선택이다.

전투 규칙으로 뽑을 점:

- 격자/위치 기반 전투
- RPG 스탯과 성장
- 전술적 의사결정이 전투의 중심

화면 구성으로 뽑을 점:

- 전투 화면에서 위치, 이동, 지형을 읽을 수 있어야 한다.
- 어디에 서야 하는지 판단할 수 없으면 턴제여도 SRPG처럼 느껴지지 않는다.

## 출처 전체에서 반복되는 패턴

반복되는 전투 규칙:

- 격자 기반 전장
- 명확한 플레이어/적 턴 또는 유닛별 턴 순서
- 행동 전 유닛 선택
- 이동 범위 표시
- 이동과 행동 선택의 분리
- 타겟 선택 전 예상 결과 표시
- 기본 공격
- 방어/대기/턴 종료 행동
- 스킬 또는 특수 능력
- 자원/쿨다운/조건 체크
- 임무 목표 또는 승리 조건 표시
- 전투 후 결과/보상 표시

반복되는 화면 요소:

- 중앙 전장 격자
- 현재 행동 유닛 표시
- 이동/공격/위험 타일 오버레이
- 턴 순서 또는 현재 페이즈 표시
- 현재 목표 텍스트
- 유닛 HP와 핵심 능력치 패널
- 타겟 미리보기 / 전투 예측
- 행동 바 또는 행동 메뉴
- 턴 종료/대기 버튼
- 전투 로그 또는 상태 메시지 영역

반복되는 피드백 상태:

- 이동 가능한 칸
- 이동 불가능한 칸
- 공격 가능한 대상
- 적 위협 범위
- 행동 비활성/조건 미충족
- 이미 행동한 유닛
- 피해 발생
- 승리/패배
