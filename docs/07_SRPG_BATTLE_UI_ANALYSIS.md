# SRPG 전투 UI 분석

작성일: 2026-06-28

입력 자료: [06_SRPG_WEB_RAW_DATA.md](06_SRPG_WEB_RAW_DATA.md)

현재 프로젝트 목표:

```text
탑다운 던전 현장 작업
-> 위험 이벤트 발생
-> SRPG식 전투 화면 진입
-> 결과를 반영하고 같은 던전 위치로 복귀
```

이 문서는 현재 전투 프로토타입을 단순 턴제 팝업이 아니라 SRPG 전투처럼 보이고 플레이되게 만들기 위해 필요한 화면 구성과 행동 흐름을 정리한다.

## 핵심 결론

SRPG 전투는 `턴제 + 격자`만으로 성립하지 않는다. 플레이어가 아래 판단 과정을 읽고 실행할 수 있어야 한다.

```text
현재 행동 유닛 확인
-> 이동 가능 범위와 위험 범위 확인
-> 이동할지 말지 결정
-> 이동 칸 선택
-> 행동 선택
-> 타겟과 예상 결과 확인
-> 실행 확정
-> 결과 처리
-> 유닛 행동 완료 표시
-> 적 턴 또는 다음 유닛 턴 진행
```

이 프로젝트는 MVP에서 조작 유닛을 보스 1명으로 유지해도 된다. 하지만 화면과 조작 흐름은 SRPG의 문법을 따라야 한다. 그렇지 않으면 플레이어는 전술 전투를 한다기보다 말판 하나를 클릭하는 느낌을 받는다.

## 최소 SRPG 요건

### 1. 전장 규칙

필수:

- 전투 맵은 고정된 격자여야 한다.
- 맵 밖으로 이동할 수 없어야 한다.
- 유닛은 칸을 점유하고 서로 겹칠 수 없어야 한다.
- 현재 행동 유닛 기준으로 이동 가능 범위가 계산되어야 한다.
- 적과 인접했을 때 선택 가능한 행동이 달라져야 한다.
- 한 턴에 가능한 행동 수가 제한되어야 한다.

다음 MVP 권장값:

| 규칙 | MVP 값 |
|---|---|
| 보스 이동 범위 | 2칸 또는 3칸 |
| 보스 기본 공격 | 인접 1칸 |
| 보스 스킬 | 2-3칸 범위, 사용 횟수 또는 쿨다운 제한 |
| 방어 | 턴 종료, 다음 피해 감소 |
| 대기 | 보너스 없이 턴 종료 |
| 도망/철수 | 전투 미해결 상태로 던전 복귀 |

### 2. 플레이어 턴 흐름

목표 흐름:

```text
플레이어 턴 시작
-> 행동 메뉴 표시
-> Move / Attack / Defend / Skill / Flee / Wait 중 선택
-> Move 선택 시 이동 가능 칸 표시
-> 이동 칸 선택
-> 행동이 남아 있으면 행동 메뉴로 복귀
-> Attack/Skill 선택 시 타겟 가능 칸 표시
-> 타겟 정보와 예상 결과 확인
-> 실행 또는 취소
-> 행동 처리
-> 조건에 따라 턴 종료
```

중요한 점:

- 이동이 곧 공격으로 이어지면 안 된다.
- 적을 클릭했다고 바로 공격하면 안 된다.
- 공격은 `Attack`을 선택한 뒤 타겟을 고르는 구조여야 한다.
- 적과 인접했을 때 공격/방어/스킬/도망/대기 선택이 의식적으로 보여야 한다.
- 도망은 승리가 아니라 전투를 해결하지 않고 나가는 행동이어야 한다.

### 3. 적 턴 흐름

MVP 적 AI는 단순해도 된다.

```text
적 턴 시작
-> 살아있는 적마다 보스와 거리 확인
-> 인접해 있으면 공격
-> 인접하지 않으면 적 이동 범위만큼 보스 쪽으로 이동
-> 이동 후 공격 가능하면 공격, 아니면 대기
-> 모든 적 처리 후 플레이어 턴으로 복귀
```

나중에 확장할 요소:

- 적 공격 사거리
- 적 스킬
- 적 위협 범위 표시
- 경보 호출, 오브젝트 파괴, 탈출 같은 목적 행동
- 적 성향 차이: 보스 공격, 시체 포대 보호/파괴, 출구 봉쇄, 도망

## 추천 화면 구성

`던전 클린업 주식회사` 전투 화면 권장 구조:

```text
+--------------------------------------------------------------+
| 턴 / 목표 / 라운드 / 의뢰 위험도                              |
+------------------------------+-------------------------------+
|                              | 적/타겟 정보 패널              |
|                              | - 이름, HP, 의도               |
|       전술 격자 맵           | - 예상 피해/결과               |
|       - 아군/적 유닛         |                               |
|       - 이동 타일            | 행동 메뉴                      |
|       - 공격 타일            | - 이동                         |
|       - 위험 타일            | - 공격                         |
|       - 목표 오브젝트        | - 방어                         |
|                              | - 스킬                         |
|                              | - 철수                         |
|                              | - 대기                         |
+------------------------------+-------------------------------+
| 보스 상태 / hover 타일 정보 / 전투 로그                       |
+--------------------------------------------------------------+
```

이 구성이 맞는 이유:

- 포지셔닝이 핵심이므로 전술 맵이 화면 중심이어야 한다.
- 오른쪽은 고정 명령 공간으로 두면 행동 버튼이 튀지 않는다.
- 상단은 지금 무엇을 해야 이기는지 계속 보여준다.
- 하단은 맵을 가리지 않고 결과와 피드백을 전달한다.

작은 프로토타입 화면에서는 아래 축약형도 가능하다.

```text
+--------------------------------------------------------------+
| 목표 + 현재 턴 상태                                           |
+--------------------------------------------------------------+
| 전술 격자 맵                                                  |
+--------------------------------------------------------------+
| 보스 HP | 타겟 미리보기 | 이동 공격 방어 스킬 철수 대기        |
+--------------------------------------------------------------+
```

## 타일/상태 시각 언어

일관된 오버레이를 써야 한다.

| 상태 | 추천 색 | 의미 |
|---|---|---|
| 선택 유닛 | 흰색 테두리 또는 점멸 마커 | 현재 행동 유닛 |
| 이동 가능 | 파랑 | 이동 가능한 칸 |
| 기본 공격 | 빨강 | 기본 공격 가능한 대상/칸 |
| 스킬 범위 | 노랑 또는 청록 | 스킬 대상/범위 |
| 적 위협 | 보라 또는 어두운 빨강 | 적이 공격할 수 있는 위험 칸 |
| 목표/상호작용 | 초록 | 보호/파괴/탈출/조작 대상 |
| 불가 | 회색/어둡게 | 보이지만 선택 불가 |

규칙:

- 현재 선택 모드와 관련된 오버레이만 강하게 보여준다.
- 이동 칸 선택 중에는 공격 하이라이트를 약하게 하거나 숨긴다.
- 공격 대상 선택 중에는 이동 하이라이트를 약하게 하거나 숨긴다.
- 비활성 행동은 버튼을 숨기지 말고 비활성으로 두며, 상태 메시지로 이유를 알려준다.

## 행동 메뉴 설계

MVP 행동 메뉴:

| 행동 | 사용 가능 조건 | 결과 |
|---|---|---|
| 이동 | 아직 이동하지 않음 | 이동 가능 칸 선택으로 전환 |
| 공격 | 행동 가능, 인접 적 존재 | 인접 적 타겟 선택 |
| 방어 | 행동 가능 | 턴 종료, 다음 피해 감소 |
| 스킬 | 스킬 준비됨, 사거리 내 대상 존재 | 스킬 타겟 선택 후 효과 적용 |
| 철수 | 시나리오가 금지하지 않음 | 전투 미해결 상태로 던전 복귀 |
| 대기 | 항상 가능 | 턴 종료 |

버튼 상태:

- 활성: 지금 사용할 수 있음.
- 비활성: 행동은 있지만 조건이 안 맞음.
- 선택됨: 해당 행동의 칸/타겟을 고르는 중.

최소 상태 메시지:

- `이동할 칸을 선택하세요.`
- `공격할 인접 대상을 선택하세요.`
- `스킬 대상을 선택하세요.`
- `방어 태세: 다음 피해가 감소합니다.`
- `철수: 전투를 해결하지 않고 던전으로 복귀합니다.`
- `대기: 턴을 넘깁니다.`

## 실행 전 보여줘야 할 정보

플레이어가 행동을 확정하기 전에 보여줘야 하는 것:

- 이동 목적지 칸
- 예상 피해량
- 공격 후 대상의 남은 HP
- 명중률, 회피율이 있는 게임이면 그 수치
- 적 반격 가능 여부
- 스킬 비용 또는 쿨다운
- 이 행동이 턴을 끝내는지 여부

현재 프로젝트 첫 SRPG 패스에서는 확률 없이 고정 피해로 가도 된다. 중요한 것은 실행 전 미리보기와 취소 가능성이다.

## 게임 테마와 연결하기

전투가 일반 판타지 체스처럼 보이면 안 된다. 청소회사 테마와 행동명을 연결한다.

| 일반 SRPG 행동 | 던전 클린업식 표현 |
|---|---|
| Attack | `제압` / 사장 권한으로 찍어누르기 |
| Defend | `버티기` / 클립보드 방패, 작업복 방어 |
| Skill | `현장 명령` / 직원 호출, 함정 작동, 청소 도구 |
| Flee | `철수` / 현장으로 물러나 전투 미해결 |
| Wait | `상황 관망` |
| Objective | 시체 포대 보호, 조사관 경보 저지, 함정 확보, 직원 도착까지 버티기 |

이렇게 해야 SRPG 규칙이 게임 설정과 붙는다.

## 권장 구현 순서

### Slice A. 진짜 행동 상태 머신

목표: 자유 클릭 전투를 명시적인 플레이어 선택 단계로 바꾼다.

상태:

```text
choose_action
select_move_tile
select_attack_target
select_skill_target
enemy_turn
victory
defeat
```

완료 기준:

- `Move`를 선택하지 않으면 아무 칸이나 클릭해서 이동할 수 없다.
- `Attack`을 선택하지 않으면 적 클릭으로 바로 공격할 수 없다.
- `Esc` 또는 우클릭으로 타겟 선택을 취소하고 행동 메뉴로 돌아올 수 있다.

### Slice B. 행동 메뉴와 버튼 상태

목표: 플레이어가 SRPG처럼 명령을 선택하게 만든다.

완료 기준:

- 이동, 공격, 방어, 스킬, 철수, 대기 버튼이 보인다.
- 버튼은 현재 조건에 따라 활성/비활성 처리된다.
- 행동을 선택하면 타일 오버레이와 상태 메시지가 바뀐다.

### Slice C. 이동 범위와 맵 경계

목표: 이동을 전술 판단으로 만든다.

완료 기준:

- 이동 가능한 칸이 표시된다.
- 맵 밖 클릭은 거부된다.
- 점유된 칸은 이동할 수 없다.
- 나중에 지형 이동 비용을 붙일 수 있는 구조다.

### Slice D. 타겟 미리보기

목표: 공격이 즉흥 클릭이 아니라 판단 후 실행되게 한다.

완료 기준:

- 적을 hover/선택하면 HP, 예상 피해, 처리 결과가 보인다.
- 인접 기본 공격과 스킬 사거리가 시각적으로 구분된다.
- 공격 결과가 전투 로그에 남는다.

### Slice E. 방어 / 철수 / 대기

목표: 공격 말고도 의미 있는 선택지를 만든다.

완료 기준:

- 방어는 다음 적 피해를 줄이고 턴을 끝낸다.
- 대기는 보너스 없이 턴을 끝낸다.
- 철수는 전투 해결 플래그를 세우지 않고 던전으로 돌아간다.

### Slice F. 목표 배너

목표: 모든 전투가 적 전멸처럼 느껴지지 않게 한다.

완료 기준:

- 전투 시작 시 목표 텍스트가 보인다.
- 승리/패배 결과가 던전 현장에 어떤 영향을 주는지 설명한다.
- 나중에 `GameState` 의뢰 데이터에서 목표 타입을 받을 수 있다.

## 다음 코드 작업의 1차 목표

우선 Slice A와 Slice B를 같이 만든다.

```text
전투 시작
-> 행동 메뉴 표시
-> 플레이어가 이동 선택
-> 이동 가능 칸 표시
-> 플레이어가 목적지 선택
-> 행동 메뉴 복귀
-> 적이 인접하면 공격 버튼 활성
-> 플레이어가 공격 선택
-> 공격 가능 타겟 표시
-> 플레이어가 적 선택
-> 피해 처리
-> 적 턴 시작
```

방어, 스킬, 철수, 대기도 같은 메뉴에 처음부터 넣는다. 효과는 단순해도 되지만, 플레이어가 전술 명령 체계를 초반부터 볼 수 있어야 한다.

## 당장 하지 않을 것

이번 패스에서는 아래를 미룬다.

- 완전한 파티 배치
- 높낮이/복잡한 지형 비용
- 명중률/치명타 RNG
- 복잡한 AI 플래너
- 전투 중 인벤토리/장비 화면
- 카메라 줌/팬 polish
- 큰 전투 로스터

이것들도 나중에는 필요할 수 있지만, 지금 빠진 핵심은 행동 선택 루프다.

## 수락 체크리스트

다음 SRPG 프로토타입은 아래를 만족해야 한다.

- 플레이어가 보이는 행동 메뉴에서 명령을 고른다.
- 이동하기 전에 이동 가능 범위가 보인다.
- 플레이어는 전투 맵 밖으로 나갈 수 없다.
- 플레이어는 공격하지 않고 방어/대기/철수할 수 있다.
- 적과 인접하면 공격 선택지가 의미 있게 활성화된다.
- 방어, 철수, 대기, 스킬이 명시적인 명령으로 존재한다.
- 피해 처리 전에 타겟 미리보기가 있다.
- 현재 턴 상태가 항상 보인다.
- 전투 목표가 보인다.
- 승리 후 같은 던전 현장 위치로 복귀한다.
